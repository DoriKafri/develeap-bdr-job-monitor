WhatsApp Integration
Send messages via wa.me deep links — no API key needed
ℹ️
How it works

WhatsApp integration uses wa.me deep links to open WhatsApp with a pre-filled message. When you click "📱 WhatsApp" on a stakeholder card, it opens WhatsApp (desktop or mobile) with the selected template pre-loaded.

✅ No API key
Works instantly with wa.me links
📱 Any device
Opens WhatsApp Desktop or mobile
📋 Templates
Pre-filled from your WhatsApp templates
1
Phone numbers from Apollo

WhatsApp buttons appear automatically on stakeholder cards when a phone number is available. Phone numbers come from Apollo.io enrichment — configure your Apollo API key in the Apollo.io tab to start enriching contacts.

2
Customize templates

Edit your WhatsApp message templates in the Templates section. Templates support all the same variables as email/LinkedIn templates ({contact_name}, {company}, {role}, etc.).

3
Queue & send

You can either send directly (click the WhatsApp button) or queue messages for batch sending. Queued WhatsApp messages appear in the Queue view with a green "📱 Send" button.

⚡

Sync & Schedule

Manage data synchronization between your integrations and the dashboard

🔄
Run Sync

Triggers the GitHub Actions workflow that syncs HubSpot CRM data and Apollo enrichment. Runs automatically every 2 hours.

📋
How it works
1 GitHub Actions runs sync_hubspot.py to fetch CRM data → writes crm_data.json
2 If Apollo is configured, runs enrich_apollo.py → writes apollo_data.json
3 Dashboard auto-loads both JSON files on page load. Data merges into job cards and the side panel.

💡 How access control works
When Google OAuth is configured with a Client ID, only users from the allowed domain can sign in. The user list below controls fine-grained roles: Admin can manage settings and users, Editor can modify templates and outreach, Viewer has read-only access to job listings. Without OAuth configured, the dashboard is open to anyone with the link.
